# Environment Strategy

## Overview

SitesBD Platform uses three primary environments for development workflow: Development, Preview (Testing/Staging), and Production.

## Environment Summary

| Environment | URL | Deploys From | Purpose |
|-------------|-----|--------------|--------|
| Development | localhost:3000-3002 | Local | Development |
| Preview | *.vercel.app | PR branches | Testing |
| Production | sites.bd, app.sites.bd, admin.sites.bd | main | Live |

## Development Environment

### Local Setup

```bash
# Clone repository
git clone https://github.com/hminhajulonline-dot/sitesbd-platform.git

# Install dependencies
npm install

# Copy environment file
cp .env.example .env.local

# Start all apps
npm run dev

# Or start individual app
cd apps/web && npm run dev
```

### Environment Variables (Development)

```bash
# .env.local
NEXT_PUBLIC_ENVIRONMENT=development
NEXT_PUBLIC_APP_NAME=SitesBD Platform
SUPABASE_URL=http://localhost:54321
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-key
```

### Ports

| App | Port | URL |
|-----|------|-----|
| Web | 3000 | http://localhost:3000 |
| Dashboard | 3001 | http://localhost:3001 |
| Admin | 3002 | http://localhost:3002 |

## Preview Environment

### Automatic Deployments

- **Trigger**: Pull request opened or updated
- **Platform**: Vercel
- **URL**: Generated by Vercel (*.vercel.app)

### Preview URLs

Each PR gets a unique preview URL:
- Web: `https://sitesbd-platform-[hash].vercel.app`
- Dashboard: `https://sitesbd-platform-dashboard-[hash].vercel.app`
- Admin: `https://sitesbd-platform-admin-[hash].vercel.app`

### Preview Environment Variables

```bash
# Set in Vercel Dashboard
NEXT_PUBLIC_ENVIRONMENT=preview
NEXT_PUBLIC_APP_NAME=SitesBD Platform (Preview)
# Supabase keys from Vercel
# Cloudflare keys from Vercel
```

### Testing on Preview

1. Open PR preview URL
2. Test functionality
3. Add comments to PR with feedback
4. Verify all three apps work

## Production Environment

### Production URLs

| App | Domain |
|-----|--------|
| Web | sites.bd |
| Dashboard | app.sites.bd |
| Admin | admin.sites.bd |

### Production Deployments

**Trigger**: Merge to `main` branch

**Process**:
1. PR merged to `main`
2. GitHub Actions runs CI
3. Vercel deploys to production
4. Health checks verify deployment

### Production Environment Variables

```bash
# Set in Vercel Dashboard > Environment Variables
NEXT_PUBLIC_ENVIRONMENT=production
NEXT_PUBLIC_APP_NAME=SitesBD Platform
# Production Supabase keys
# Production Cloudflare keys
```

## Environment Variable Management

### Hierarchy

1. `.env.example` - Template (committed)
2. `.env.local.example` - Local dev template (committed)
3. `.env.local` - Local overrides (not committed)
4. Vercel Dashboard - Environment-specific values

### Required Variables by Environment

| Variable | Development | Preview | Production |
|----------|-------------|---------|------------|
| NEXT_PUBLIC_ENVIRONMENT | ✓ | ✓ | ✓ |
| NEXT_PUBLIC_APP_NAME | ✓ | ✓ | ✓ |
| SUPABASE_URL | ✓ | ✓ | ✓ |
| SUPABASE_ANON_KEY | ✓ | ✓ | ✓ |
| SUPABASE_SERVICE_ROLE_KEY | ✓ | ✓ | ✓ |
| CLOUDFLARE_API_TOKEN | ✗ | ✓ | ✓ |
| CLOUDFLARE_ZONE_ID | ✗ | ✓ | ✓ |
| SMTP_HOST | ✓ | ✓ | ✓ |
| SMTP_PORT | ✓ | ✓ | ✓ |
| SMTP_USER | ✓ | ✓ | ✓ |
| SMTP_PASSWORD | ✓ | ✓ | ✓ |
| CRON_SECRET | ✗ | ✓ | ✓ |
| API_SECRET | ✗ | ✓ | ✓ |

### Security Rules

1. **Never commit**: `.env`, `.env.local`, `*.local.ts`
2. **Always template**: `.env.example`, `.env.local.example`
3. **Restrict in Vercel**: Service keys only on server-side
4. **Public prefix**: Only `NEXT_PUBLIC_*` vars safe for browser

### Adding New Variables

1. Add to `.env.example`
2. Add to `.env.local.example`
3. Add to appropriate Vercel environment
4. Document in this file

## Switching Environments

### In Code

```typescript
const environment = process.env.NEXT_PUBLIC_ENVIRONMENT;

if (environment === 'production') {
  // Use production config
} else if (environment === 'preview') {
  // Use preview config
} else {
  // Use development config
}
```

### Feature Flags

```typescript
const isProduction = process.env.NEXT_PUBLIC_ENVIRONMENT === 'production';
const showBetaFeatures = !isProduction;
```

## Troubleshooting

### Environment Not Loading

1. Check `.env.local` exists
2. Restart dev server
3. Clear `.next` cache: `rm -rf .next`

### Variable Not Found

1. Check spelling matches in `.env`
2. Restart application
3. Verify Vercel environment variable set

### Build Fails on Production

1. Check all required variables set in Vercel
2. Verify Supabase/Cloudflare keys are valid
3. Review Vercel deployment logs
